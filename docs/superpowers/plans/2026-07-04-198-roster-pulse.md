# Roster row pulses for the acting role (#198) Implementation Plan

> Small pure-frontend slice; single file. REQUIRED SUB-SKILL: superpowers:executing-plans.

**Goal:** In the REPOS & ROLES roster, the role row of the role that is *acting right now* gets a subtle pulse/highlight, tying the Handoffs feed (#197) to the roster.

**Architecture:** Pure `lib/dashboard_page.html` change consuming existing state — no `dashboard_state.py`/backend change. The "acting" signal is the LIVE session's role: `r.current_session.role` while `dispStatus(r)==="working"` — the exact thing the supervisor writes as the `session-running $role` heartbeat phase (`supervisor.sh:997`) and the focus card already shows (`cs.role`). Gating on `working` means a stale `current_session` on an idle/stopped repo never pulses (per #177 non-goal: no fake motion). A subtle green pulsing dot + role-name accent, reusing the existing `@keyframes pulse`.

**Tech Stack:** vanilla JS + CSS in one file; verified via chrome-devtools.

## Global Constraints

- Read-only/dashboard-only (same posture as #197); no new data, no network.
- No fake motion — the pulse is backed by a real live state (a streaming session), never a timer or a stale field.
- Keep it subtle; healthy = boring. Semantic color only on the abnormal/active state.
- Don't regress the #164 identical-markup rebuild-skip: acting on/off changes the markup (so it rebuilds and the pulse appears/disappears), but while acting persists the markup is identical (no rebuild → the CSS animation keeps running, no restart flicker).

- [ ] **Step 1: Add `actingRole`** near `dispStatus` (~line 356):

// #198: who is acting RIGHT NOW = the live session's role (== the supervisor's
// `session-running <role>` heartbeat phase). Only while actually working, so a
// stale current_session on an idle/stopped repo never pulses (no fake motion).
function actingRole(r){
  return dispStatus(r)==="working" ? ((r.current_session||{}).role||"") : "";
