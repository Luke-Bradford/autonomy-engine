# W2 Event bus (event triggers) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development (or
> superpowers:executing-plans) to implement task-by-task. **Codex checkpoint 1 on this plan before
> execution** (settled-decision #19). Steps use checkbox (`- [ ]`) tracking.

**Goal:** Fire `trigger.type: event` roles when board/PR state changes, by folding a gh-poll event
check into the supervisor's loop iteration (after the W1 cron check), reusing `run_session` and the
lifetime-held per-repo lock. No new process, no new lock, no inbound network. Unblocks the
Coder→QA choreography (QA reacts to `pr.opened`/`pr.synchronize`).

**Spec:** `docs/superpowers/specs/2026-07-03-event-bus-design.md`. Issue: #86. Depends on W1 (#85,
merged). Branch: `feat/86-event-bus`.

**Architecture (from the spec):** `roles.py events <repo>` enumerates event roles + their `on:`
lists; the supervisor polls gh once per iteration, compares against supervisor-owned per-event
cursors under `$VARDIR/events/`, and fires listening roles through `run_session` (at-least-once:
advance the cursor AFTER dispatch). Event roles become dispatchable exactly as cron roles did in W1.

## Global constraints (CI-enforced)

- macOS `/bin/bash` 3.2.57: NO `mapfile`/`readarray`, NO globstar, NO `declare -A`, NO `${var,,}`.
  The `pr.synchronize` per-PR SHA map is a `NUMBER<TAB>SHA` flat file, NOT an associative array.
- Python 3 **stdlib only**; config parsing only via `lib/config_parser.py`.
- `shellcheck -S warning` clean: `start bin/*.sh bin/agents/*.sh tests/*.sh templates/autonomy-pack/qa/*.sh`.
- Tests source the real scripts / import the real modules; stub only the `run_session` / `gh` /
  `roles.py events` seams (the seams `test_scheduler.sh` + `test_headless_dispatch.sh` established).
- **Fail-safe never fail-open:** a gh-poll failure fires nothing and leaves cursors + loop dispatch
  byte-for-byte unchanged; a cursor-write failure leaves the cursor so the event re-delivers
  (at-least-once), never crashes the loop.
- **Sole-writer invariant:** only the supervisor writes `$VARDIR/events/*` (reset-epoch split /
  cron-marker, generalised — adapters never persist event state).
- **Dispatchability (W1 lesson, do not regress):** event roles MUST be in `dispatchable_roles` so
  `run_session` resolves them; W1's Codex checkpoint 2 caught cron roles being inert because
  `role_settings` refused non-loop roles — the identical trap for events.
- **Config-sourced strings re-validated at point of use (prevention-log #6):** an `on:` token is
  matched with a `case`; an unknown token WARNs and is ignored, never executed. The role name is
  charset-gated before any path/log use.
- No regression: with no event roles, the loop behaves byte-for-byte as today (and as post-W1).

### Task 2: supervisor gh-poll → cursor delta → fire → advance

**Files:** Modify `bin/supervisor.sh`; Create `tests/test_event_bus.sh`.

**Interfaces:**
- Consumes: Task 1's `roles.py events`; existing `run_session`, `$VARDIR`, `gh`, the `log` helper,
  the role-name/`on:`-token charset idioms.
- Produces: `resolve_event_wakes` fires each event role whose `on:` matched a new event since its
  cursor, advancing the cursor AFTER dispatch (at-least-once); first-sight initialises cursors
  without firing; poll/dispatch failure re-delivers or skips without touching loop dispatch.

- [ ] **Step 1: Failing test** in `tests/test_event_bus.sh` (source `bin/supervisor.sh`; `VARDIR` →
  temp; stub `run_session` to a capture file; factor the gh poll behind a seam function the test
  overrides, e.g. `_event_poll <event>` printing the current cursor value(s)):
  - a new `pr.opened` (poll reports PR #5, cursor at #4) **fires** the listening role once and the
    role's `<role>__pr.opened.cursor` advances to #5.
  - the same PR on the next tick (cursor now #5) does **not** re-fire.
  - a **first-sight** event (no cursor) initialises the cursor without firing (baseline reset).
  - `run_session` failing (rc!=0) leaves that role's cursor unadvanced → the event **re-delivers**
    to that role next tick (at-least-once); a *second* listener whose fire succeeded keeps its own
    advanced cursor (per-`(role,event)` isolation — no cross-listener loss).
  - `_event_poll` failing (rc!=0) → `resolve_event_wakes` returns 0, fires nothing, cursor untouched.
  - `session.done` fires once for a loop/cron session that ran this tick, and an event-phase session
    does NOT re-trigger it (reentrancy guard).
- [ ] **Step 2:** run `bash tests/test_event_bus.sh` — expect FAIL.
- [ ] **Step 3: Implement** in `bin/supervisor.sh`:
  - `_event_poll <event>` wraps the per-event `gh` query (table in the spec; `--state all --limit
    200` for number-keyed events). rc!=0 → caller skips that event. JSON via a stdlib-`json` Python
    one-liner (no `jq`).
  - `resolve_event_wakes <session_ran>`: enumerate event roles via `roles.py events`;
    `mkdir -p "$VARDIR/events"`; for each `(role, event)` a role listens for: charset-gate the role
    name; `session.done` uses the passed-in `<session_ran>` per-tick edge (no poll); otherwise read
    `$VARDIR/events/<role>__<event>.cursor` (first-sight → write current max/state, continue),
    compute the delta vs `_event_poll`, and if non-empty `run_session <role>` — on success advance
    that role's cursor (at-least-once); on failure leave it (re-deliver). Prune `pr.synchronize.seen`
    to the currently-open PR set and `merge.done.seen` to the most-recent `--limit` merged numbers
    (bounded windows aligned with the poll page — never drop a still-pollable item and re-deliver it).
    `case`-match `on:` tokens (unknown → WARN, skip — defense-in-depth behind validate_roles). Never
    `exit` non-zero.
  - `main` computes `session_ran` from whether the loop/cron phase dispatched a session this tick and
    passes it in; event-phase sessions do not set it (guard).
  - Call `resolve_event_wakes` once per loop iteration in `main`, after `resolve_cron_due`, inside
    the held lock.
- [ ] **Step 4:** run `bash tests/test_event_bus.sh && bash tests/test_scheduler.sh &&
  bash tests/test_headless_dispatch.sh` — expect PASS + no regression to cron / loop dispatch.
- [ ] **Step 5: shellcheck** `bin/supervisor.sh tests/test_event_bus.sh` — expect no output.

## Self-review notes

- Spec coverage: enumeration + dispatchability → Task 1; poll + cursor + fire + advance → Task 2;
  wire-through/docs → Task 3. Reuses `run_session` + the W1 marker pattern; no new lock/process.
- Invariants: sole-writer cursors (Task 2) = reset-epoch split generalised; one-session-per-iteration
  preserved (events fire under the held lock, sequential with loop + cron); fail-safe (poll failure

  deliberate inverse of cron's under-fire, justified in the spec); charset re-validation of the role

- No-regression: Task 2 keeps `test_scheduler.sh` + `test_headless_dispatch.sh` green (cron + loop
  dispatch untouched when no event roles / when the poll fails).

  NOT `.autonomy/**`, `bin/safe_merge.sh`, or `.github/workflows/**`. The self-pack `qa` event rails
