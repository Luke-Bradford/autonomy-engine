# BYO-LLM Implementation Plan (sub-project 1)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let a `roles:` agent run against a local (or any) OpenAI-compatible LLM endpoint — a new
`openai_compatible` account kind + per-role agent type + the codex adapter honoring
`OPENAI_BASE_URL` + a background-`prep` role on a local model — so cheap groundwork offloads to a
free local model and saves cloud tokens. Repo ships the plug-in, not the model.

**Architecture:** Extends increment 3's dispatch (role's `account:` → `accounts.py resolve` → env
in the session subshell → adapter). A new account kind resolves to `OPENAI_BASE_URL` (+ key); a new
per-role `agent:` lets one repo run `coder` on `claude` (cloud) and `prep` on `codex` (local); the
codex adapter, seeing `OPENAI_BASE_URL`, points codex at the endpoint via its native local-provider
support. Everything degrades to today's behaviour when the new fields are unset.

**Tech Stack:** bash 3.2.57, Python 3 stdlib only, `unittest` + the source-the-real-script bash
test harness, Ollama (env only, not in repo).

**Spec:** `docs/superpowers/specs/2026-07-02-byo-llm-design.md`. Issue: #78. Branch: `feat/78-byo-llm`.

## Global Constraints (CI-enforced)

- macOS `/bin/bash` 3.2.57: NO `mapfile`/`readarray`, NO globstar, NO `declare -A`, NO `${var,,}`.
- Python 3 **stdlib only**; config parsing only via `lib/config_parser.py`.
- `shellcheck -S warning` clean: `start bin/*.sh bin/agents/*.sh tests/*.sh templates/autonomy-pack/qa/*.sh`.
- Tests source the real scripts / import the real modules; stub only at the boundary (Keychain via
  the injected credentials fake; the local HTTP endpoint via a stub responder or the `curl` seam).
- **Fail-safe never fail-open:** a malformed/unresolvable endpoint REFUSES (raises / returns
  non-zero), never runs against a broken endpoint.
- **No secrets in the accounts index** (names/kinds/labels/base_url only, mode 600, atomic writes).
- **No regression:** the existing cloud adapter paths behave byte-for-byte the same when
  `OPENAI_BASE_URL` is unset and when no per-role `agent:` is set.
- Repo-agnostic `bin/`/`lib/`; TDD (failing test first); frequent commits.

def _valid_base_url(url):
    """A well-formed http(s) URL with a host -- the endpoint a role calls."""
    try:
        p = urllib.parse.urlparse(url or "")
    except (ValueError, AttributeError):
        return False
    return p.scheme in ("http", "https") and bool(p.netloc)

```python
def _parse_models_payload(raw):
    """Model ids from an OpenAI /v1/models JSON body. [] on any problem
    (best-effort -- the config UI degrades to manual entry)."""
    try:
        data = json.loads(raw)
        rows = data.get("data")
        if not isinstance(rows, list):
            return []
        return [r["id"] for r in rows
                if isinstance(r, dict) and isinstance(r.get("id"), str)]
    except (ValueError, TypeError, KeyError):
        return []
```

```python
    def list_models(self, name):
        """Model ids the endpoint advertises (GET <base_url>/models). []
        on any error -- best-effort discovery, never raises."""
        entry = self.get(name)
        if not entry or entry.get("kind") != "openai_compatible":
            return []
        base = (entry.get("base_url") or "").rstrip("/")
        try:
            req = urllib.request.Request(base + "/models")
            label = entry.get("credential")
            secret = self.credentials.get_secret(label) if label else None
            if secret:
                req.add_header("Authorization", "Bearer %s" % secret)
            with urllib.request.urlopen(req, timeout=5) as resp:
                return _parse_models_payload(resp.read())
        except (urllib.error.URLError, OSError, ValueError):
            return []
```
